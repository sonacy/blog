前端搬砖之路

自我介绍

大学

  • 考试全靠小超市买往年试题，考前一周一把梭，成绩整体还不错

  • 没有参加过学生会和社团，但是交了不少盆友，包括南财，南师大，南中医

  • 学校附近都玩过，最爱大成，大坑 小吃美食，金鹰也常去，周边酒吧都跟老板混熟了 😳

  • 跟代码的交道就那几门课，那时对前端的理解那时候就是 0

工作前三年

  • 大学应聘上的公司：华为，电信，苏宁。但都是网络工程相关。

  • 专业是通信工程，而且家里人比较支持国企, 所以就去了电信。

  • 那时候南京没啥互联网氛围，现在的啥 bat，字节，美团，滴滴啥的都没有

  • 国企的印象: 轻松压力小，但也没啥空间发展，对于年轻员工的激励也基本没有

  • 其实本质也在于我们并没有赶上通信行业福利好的那几年

转机

  • 当时还是挺好玩，因为自己赚钱了，可选择空间更大，但同时也出了点意外，也因为这自己有时间静下心来思考了

  • 当时我自我评价自己还是比较聪明的，学习能力很强，而且对科技还是比较感兴趣的。二是选对方向行业很重要，那时候的互联网很红火，程序员对我来说就是最好选择了

  • 开始了自学 java，这个比较主流，网上教程也较多，再加上大学时学的一些基础课，所以基础不错就很快掌握了

  • 电信那会公司找了个外包，做了个针对网络设备的管理系统，当然这个项目最后黄了，人家尾款没有收回。自己对项目比较熟，就自己用 java 的 ssh 框架搞了个，也算个全栈了，后面还搞了 Android，给系统搞了个安卓移动版本。因为这个还被领导升级了

  • 这时候对于前端的理解就是 jquery，以及很多 jquery 的插件。

转岗

  • 当时已经准备离职去互联网行业，然后领导不同意，提出了电信也有相关的领域，涨了薪换了部门, 自己傻傻的接受了

  • 然而感觉在电信更多的还是叫外包工作多，自己写代码，自己搞架构少，发展的空间还是较小

  • 这一年让我更加自由的学习 coding 了，搞了 python，完了 scala，搭了 hadoop，spark 集群，知道了啥是函数式编程，又去学了 lisp，haskell 等比较偏门的语言

  • 当然这一年也接触了 vue react angularjs 这些东西, 知道了 nodejs

  • 针对公司的一些 h5 宣传页，做了基于 react ssr 的改造

  • 这时候我对前端的理解加大了很深，知道了前端再不是简单的切图仔，知道了前端框架，知道了 es5,知道了前后端分离，知道前端工程化，也确定了自己未来的方向

过程

  • 当时目标是阿里，但也不太想离开南京。。。所以觉得先找个中转站，就去 vivo。

  • 在那里主要是做个基于 vue 的搭建系统，用于 h5 宣传页。除了基本业务，最主要就是把原来通过 node 走的 http 实现的编译能力直接转移到浏览器端了，搞了个vue live compiler

  • 待了一个多月，不知道是因为刚从国企离开的不适应，还是那里的文化氛围还是不太适合我，我就离开了，去了南京阿里

  • 那时候对于前端再次加深，做了搭建，搞了简单的编译。

南京阿里

  • 在阿里经常去杭州出差，当时前端的 leader 是徐飞，也许很多人看过他的 blog，就是民工叔, 以前也来南邮分享过。

  • 当时他也经常请很多业界的大佬们来交流分享下技术，经过这一波其实认识了不少业界的大佬们。

  • 在阿里最主要接触技术在于三点，一是多端统一（一套代码生成 web，native，小程序），二是 web 端 ide，三是 GraphQL

  • 因为一些公司架构调整的等原因，一年多后我也离开了

  • 这段时间的前端技术提升是很快，技术视野也很大的提高了，当时做的那三点都属于业界比较前沿的技术了

字节生活

  • 我在部门是字节的数据平台

  • 字节的氛围更加宽松，技术上不设边界，技术氛围较强，空间较大，身边也有很多业界优秀的人可以一起学习，大家也乐于一起讨论，交流

  • 技术上一开始就遇到了 js 转 ts 的阶段，这里直接利用的一些 ts 自己的 compiler 的能力，做了个 codemod，解决一些重复枯燥的工作

  • 为了保证质量，研究了下业界的 e2e 的测试方案，根据我们的业务，以及 chrome 的一些底层实现，搞了一套更适合我们业务的 e2e 测试方案

  • 在字节跟在阿里的体会不同点在于，阿里的一些技术方向工作还是 leader 给与，自己研究然后开发。在字节更多的是自己思考，自己给方案，自己去推广。空间和发展空间更大。

  • 前端的发展：其实这时候已经不局限于前端领域，而是可以以更高的角度去看待工程，思考现有的不足，尝试找到合适的解决方案。

自我总结下

  • 大学期间最重要还是知道自己想要啥，喜欢啥，多尝试，找到后努力让自己拥有自由选择的能力

  • 程序员还是比较好入职的，不管大家现在的专业是啥，后面我们也会提供很多的资料给大家学习

  • 大多数程序员没有达到比拼天赋的境界上，找清楚方向，足够努力，一定能有所收获的

  • 选对行业方向很重要，会让自己少奋斗很多年

  • 越早入行机会越多，优势越大

  • 都说程序员工作年限很短，这说明几个点，行业更新很快，也意味着机会更多。如果是我后面做的一些专有方向并且深入很多的话，很难被取代。
